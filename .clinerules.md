# Cline Rules Priority — Roadmap Sync Contract

This file is the **priority reference for Cline** for roadmap synchronization governance.
If any rule conflicts with other local instructions, this file takes precedence for ROADMAP_SYNC behavior.

## ROADMAP_SYNC (website) — Mandatory PR contract

Every pull request description MUST include this exact metadata block:

## Roadmap Sync (website)

<!-- ROADMAP_SYNC_START -->

user_facing: yes|no
phase: 1-10
impact_type: feature|fix|security
summary_fr: <1-2 lignes>
summary_en: <1-2 lignes>
evidence_links: <comma-separated urls>

<!-- ROADMAP_SYNC_END -->

## Validation rules

- Block markers are mandatory and must be unique.
- Keys are mandatory and must stay in strict order.
- `user_facing` must be `yes` or `no`.
- `phase` must be an integer between 1 and 10.
- `impact_type` must be one of `feature`, `fix`, `security`.
- `summary_fr` and `summary_en` must be real content (no placeholders).
- `evidence_links` must contain at least one valid `http(s)` URL.

## Label sync behavior

- If `user_facing: yes`, the PR must have label `roadmap:user-facing`.
- If `user_facing: no`, label `roadmap:user-facing` must be removed.

## Dispatch behavior

A roadmap dispatch to `benoit-bremaud/brasse-bouillon-website` is allowed only when:

1. the PR is merged,
2. the base branch is `main`,
3. `user_facing` is `yes`,
4. metadata validation passes.

Dispatch event contract:

- `event_type`: `roadmap_user_facing_update`
- secret: `WEBSITE_DISPATCH_TOKEN`
- `client_payload` keys:
  - `source_repo`
  - `pr_number`
  - `pr_url`
  - `pr_title`
  - `merged_at`
  - `phase`
  - `impact_type`
  - `summary_fr`
  - `summary_en`
  - `evidence_links`
